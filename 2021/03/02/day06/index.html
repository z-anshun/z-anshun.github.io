访问权限

  1. private：私有访问权限，只能在本类中进行访问
  2. default：默认访问权限，可在本包中进行访问
  3. protected：保护访问权限，只能被本包和不同包的子类访问
  4. public：公开访问权限，所以类中，均可访问

命名规范

再次重提，回顾前面

  1. 类：单词首字母大写
  2. 方法：第一个单词小写，之后的每个单词都大写，如getInfo()
  3. 属性：跟方法一样，如：studentName
  4. 包：所有单词均小写
  5. 常量：所以单词字面均大写

解释，继承实际上是是重写了 run() 方法，由于线程的资源和 Thread 实例捆绑在一起，所以不同的线程的资源不会进行共享。

其实这里也不难理解，因为本质上调用的都是同一个继承的对象。这种实现方式就是静态代理的方式，线程资源与 Runable 实例捆绑在一起，Thread 只是作为一个代理类，所以资源可以进行共享

因此，我们一般使用实现Runnable的接口来实现线程的调用

这样做的优势有：

  1. 实现 Runnable 大多数情况下是比继承 Thread 更好的方法。
  2. Java 只支持单继承，所以你继承了 Thread 的话，其他的类你就不能继承了。
  3. 如果实现 Runnable 接口，多线程可以直接将你的工作并发处理（直接使用 start）。而继承 Thread 不能直接进行多线程工作，你得为你的每一部分工作都定义一个线程。
  4. 其他的一些API，接口都是 Runnable 类型的。比如 Java 内置的线程池 API ExcutorService 等。
  5. 其实 Thread 只是实现 Runnable 接口，并增加了一些常用方法而已

解释，为什么说单继承的局限性？

因为继承Thread，我们相当于只是继承了Thread，没有实现Runnable的多线程功能，也就是现在是确保每部分工作都在一个独立的线程，即你对一个对象调用两次start是错误的

线程状态

线程分为，创建、就绪、运行、堵塞和死亡5个状态

  1. 创建：即new Thread
  2. 就绪：当我们调用start()方法时，该线程就会进行就绪状态，遵循CPU的调度
  3. 运行：当就绪状态的线程被调度并获取到处理器资源时，此时便会调用该线程对象的run()方法
  4. 阻塞：调用sleep()、wait()、suspend()等方法时，线程便会被阻塞，即无法进入排队队列
  5. 死亡：即调用结束或者调用stop()方法终止了进程

方法名 类型 描述
public Thread(Runnable target) 构造 接收Runnable接口的子类对象，实例化Thread对象

设置为守护线程：

  1. 优先级低
  2. 不能操作文件、数据库等资源，避免主线程关闭而未能关闭守护线程的资源
  3. 它会在任何时候甚至在一个操作的中间发生中断
  4. 主线程关闭后无需手动关闭守护线程，因为会自动关闭
  5. Java垃圾回收线程就是一个典型的守护线程，
  6. 也可以理解为所有为线程服务而不涉及资源的线程都能设置为守护线程
